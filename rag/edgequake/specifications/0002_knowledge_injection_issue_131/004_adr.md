# ADR-0002: Knowledge Injection for Enhanced Search

**Issue**: [#131](https://github.com/raphaelmansuy/edgequake/issues/131)
**Status**: Proposed
**Decision**: Candidate A — Tagged Ingestion
**Date**: 2026-04-02

---

## Context

Domain-specific workspaces need enrichment with acronym definitions, synonym
mappings, and contextual glossaries. These definitions must improve retrieval
quality but must NOT appear as cited sources in query responses.

## Decision

### API Design

```
PUT /api/v1/workspaces/:workspace_id/injection
  Body: { "content": "OEE = Overall Equipment Effectiveness\n..." }
  Auth: Admin or workspace owner
  Semantics: Replace (upsert). New content replaces previous version.
  Response: 202 Accepted
  {
    "injection_id": "uuid",
    "workspace_id": "uuid",
    "version": 3,
    "status": "processing",
    "previous_version": 2
  }

GET /api/v1/workspaces/:workspace_id/injection
  Response: 200
  {
    "injection_id": "uuid",
    "content": "OEE = ...",
    "version": 3,
    "status": "completed",
    "entity_count": 15,
    "created_at": "...",
    "updated_at": "..."
  }

DELETE /api/v1/workspaces/:workspace_id/injection
  Response: 204 No Content
  Side effects: Remove all injection entities, vectors, chunks
```

### Data Flow

```
+------------------------------------------------------------------+
|               INJECTION LIFECYCLE                                |
|                                                                  |
|  PUT .../injection { content: "..." }                            |
|    |                                                             |
|    v                                                             |
|  1. Generate stable document ID                                  |
|     doc_id = "injection::{workspace_id}"                         |
|    |                                                             |
|    v                                                             |
|  2. If previous injection exists:                                |
|     CASCADE DELETE by source_document_id = doc_id                |
|     - KV: delete doc + chunks                                    |
|     - Vector: delete by metadata.source_document_id              |
|     - Graph: delete nodes/edges by source_id containing doc_id   |
|    |                                                             |
|    v                                                             |
|  3. Process through standard pipeline:                           |
|     Pipeline::process(doc_id, content)                           |
|     - Chunking (same config as regular docs)                     |
|     - Entity extraction (same LLM)                               |
|     - Embedding generation (same model)                          |
|    |                                                             |
|    v                                                             |
|  4. Tag all artifacts:                                           |
|     - KV doc metadata: { "source_type": "injection" }            |
|     - KV chunk metadata: { "source_type": "injection" }          |
|     - Vector metadata: { "source": "injection" }                 |
|     - Graph node property: { "source_type": "injection" }        |
|    |                                                             |
|    v                                                             |
|  5. Merge into KG (normal merge semantics):                      |
|     - Injection entity "OEE" merges with doc-extracted "OEE"     |
|     - Description becomes richer                                 |
|     - source_id accumulates injection + doc chunks               |
|    |                                                             |
|    v                                                             |
|  6. Update workspace metadata:                                   |
|     metadata.injection_version += 1                              |
|     metadata.injection_updated_at = now()                        |
+------------------------------------------------------------------+
```

### Citation Exclusion at Query Time

```
+------------------------------------------------------------------+
|               QUERY CITATION FILTERING                           |
|                                                                  |
|  Query: "What is the OEE of Line 3?"                            |
|    |                                                             |
|    v                                                             |
|  Vector search returns chunks from:                              |
|    - doc-001 (manufacturing report)                              |
|    - injection::workspace-xyz (glossary)  <-- tagged             |
|    - doc-003 (operational data)                                  |
|    |                                                             |
|    v                                                             |
|  Graph traversal finds OEE entity with:                          |
|    - description: "Overall Equipment Effectiveness..."           |
|    - source_id: "injection-chunk-1|doc-001-chunk-5"              |
|    |                                                             |
|    v                                                             |
|  LLM receives ALL context (including injection)                  |
|  for richer answer generation                                    |
|    |                                                             |
|    v                                                             |
|  Citation builder FILTERS sources:                               |
|    cited_sources = sources.filter(s => s.source != "injection")  |
|    |                                                             |
|    v                                                             |
|  Response:                                                       |
|    answer: "Line 3 OEE is 87.3% ..."                            |
|    sources: [doc-001, doc-003]  <-- NO injection source          |
+------------------------------------------------------------------+
```

### Replace Semantics (Key Constraint)

Per the issue: "It is assumed that there is only one enrichment document.
Any new version cancels and replaces the previous one."

This is enforced by:

1. Using a deterministic `doc_id = "injection::{workspace_id}"`.
2. Cascade deleting all artifacts with that `source_document_id` before re-processing.
3. No version history (only current version stored).

## Consequences

### Positive

- Injection enriches both vector search AND graph traversal
- Zero changes to existing query modes (filter is additive)
- Clean replacement semantics via cascade delete
- Entities from injection merge with doc-extracted entities

### Negative

- Re-processing injection costs LLM tokens (entity extraction)
- Large glossaries (>50KB) may produce many chunks
- Merged entity descriptions may become noisy if injection conflicts with docs

### Mitigations

- Short glossary entries produce focused entities (recommended format)
- Character limit on injection content: 100KB max (v1)
- Admin can delete injection to revert

## Anticipated Roadblocks

| Roadblock                            | Resolution                                                            |
| ------------------------------------ | --------------------------------------------------------------------- |
| Cascade delete by source_document_id | Already supported: `delete_document` removes chunks/vectors by doc_id |
| Pipeline needs `source_type` flag    | Add optional `ProcessingOptions` struct to `Pipeline::process()`      |
| Query citation filter                | Add condition in `query/src/builder.rs` on source metadata            |
| Graph node property filter           | Add `source_type` to node properties in merger                        |
| Vector metadata filter               | Already supports arbitrary metadata keys                              |

## Definition of Done

- [ ] `PUT /workspaces/:id/injection` accepts text, processes through pipeline, tags artifacts.
- [ ] `GET /workspaces/:id/injection` returns current injection content and stats.
- [ ] `DELETE /workspaces/:id/injection` removes all injection artifacts.
- [ ] Injection entities appear in graph traversal during queries.
- [ ] Injection chunks contribute to vector search context.
- [ ] Injection sources are EXCLUDED from query response `sources` list.
- [ ] Submitting new injection replaces previous (cascade delete + re-process).
- [ ] Injection content limited to 100KB.
- [ ] Workspace metadata tracks injection version.
- [ ] All existing query tests still pass.

## Test Stories

### TS-001: Create injection

```
GIVEN  workspace "manufacturing" with 0 injections
WHEN   admin PUT /workspaces/:id/injection { "content": "OEE = Overall Equipment Effectiveness\nMTBF = Mean Time Between Failures" }
THEN   response 202 with version=1, status="processing"
AND    after processing: OEE and MTBF entities exist in graph
AND    vector store contains injection chunks with source="injection"
```

### TS-002: Injection improves query results

```
GIVEN  workspace has document about "Line 3 OEE was 87.3%"
AND    injection defines "OEE = Overall Equipment Effectiveness"
WHEN   user queries "What is OEE?"
THEN   response includes explanation of Overall Equipment Effectiveness
AND    sources list does NOT include the injection document
AND    sources list includes the Line 3 document
```

### TS-003: Replace injection

```
GIVEN  workspace has injection v1 with "OEE = ..."
WHEN   admin PUT /workspaces/:id/injection { "content": "OEE = ...\nSPC = Statistical Process Control" }
THEN   response 202 with version=2
AND    old entities from v1 are deleted
AND    new entities from v2 are created
AND    SPC entity exists in graph
```

### TS-004: Delete injection

```
GIVEN  workspace has injection v2 with OEE and SPC entities
WHEN   admin DELETE /workspaces/:id/injection
THEN   response 204
AND    injection entities removed from graph (unless also from docs)
AND    injection vectors removed
AND    GET /workspaces/:id/injection returns 404
```

### TS-005: Injection entity merges with document entity

```
GIVEN  document contains entity "OPENAI" with description "AI company"
AND    injection says "OPENAI = OpenAI, a leading AI research lab"
WHEN   injection is processed
THEN   OPENAI entity has merged description from both sources
AND    OPENAI source_id contains both doc chunk and injection chunk
AND    queries citing OPENAI only show the document source
```

### TS-006: Content size limit

```
GIVEN  any workspace
WHEN   admin PUT /workspaces/:id/injection with 200KB content
THEN   response 413 "Injection content exceeds 100KB limit"
```
